Blocks are the building units of a Mercur 2.0 marketplace. Each block is a self-contained piece of functionality — a module, workflow, API route, link, or UI extension — that gets copied directly into your project.

## Block types

<CardGroup cols={3}>
  <Card title="Modules" icon="database">
    Data models and business logic. Define entities, services, and database schemas.
  </Card>
  <Card title="Workflows" icon="arrow-progress">
    Multi-step business processes with compensation logic and extension hooks.
  </Card>
  <Card title="API Routes" icon="route">
    HTTP endpoints for Admin, Vendor, and Store APIs.
  </Card>
  <Card title="Links" icon="link">
    Relationships between modules that enable cross-domain queries.
  </Card>
  <Card title="Admin Extensions" icon="table-columns">
    Widgets and pages for the admin dashboard.
  </Card>
  <Card title="Vendor Extensions" icon="store">
    Widgets and pages for the vendor portal.
  </Card>
</CardGroup>

## Installing blocks

```bash
# Search for blocks
npx @mercurjs/cli@canary search

# Add specific blocks
npx @mercurjs/cli@canary add reviews commission

# Check for upstream changes
npx @mercurjs/cli@canary diff reviews
```

## Available blocks

| Block | Type | Description |
|---|---|---|
| `reviews` | Module + Workflows + API | Product and seller review system |
| `commission` | Module + Workflows | Commission rules and calculations |
| `requests` | Module + Workflows + API | Product moderation and approval flows |
| `order-return-request` | Module + Workflows | Return request escalation handling |
| `wishlist` | Module + API | Customer wishlists |
| `payout` | Module + Workflows | Vendor payout processing |
| `attributes` | Module + Links | Custom product attributes |
| `secondary-categories` | Module + Links | Alternative category trees |
