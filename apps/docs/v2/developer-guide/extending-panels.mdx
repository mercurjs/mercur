Both the admin panel and vendor portal use the same SDK (`@mercurjs/dashboard-sdk`). Customization is file-based — you add pages by creating files and control navigation through exports.

## How dashboard mounting works

Dashboard registration is backend-first. The backend (`packages/api/medusa-config.ts`) controls which app is mounted and where:

```typescript packages/api/medusa-config.ts
import path from 'path'

modules: [
  {
    resolve: '@mercurjs/core-plugin/modules/admin-ui',
    options: {
      appDir: path.join(__dirname, '../../apps/admin'),
      path: '/dashboard',
    },
  },
  {
    resolve: '@mercurjs/core-plugin/modules/vendor-ui',
    options: {
      appDir: path.join(__dirname, '../../apps/vendor'),
      path: '/seller',
    },
  },
]
```

Each panel app bootstraps by pointing the dashboard SDK at the backend config:

```typescript apps/admin/vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { mercurDashboardPlugin } from '@mercurjs/dashboard-sdk'

export default defineConfig({
  plugins: [
    react(),
    mercurDashboardPlugin({
      medusaConfigPath: '../../packages/api/medusa-config.ts',
    }),
  ],
})
```

The SDK reads `medusa-config.ts` at build time to derive `virtual:mercur/config` — there is no hand-authored config file in the panel app.

## Adding pages

Create a `page.tsx` file inside `src/pages/` and export a default React component. The route is determined by the file path.

```tsx src/pages/reviews/page.tsx
import { Star } from "@medusajs/icons"
import type { RouteConfig } from "@mercurjs/dashboard-sdk"

export const config: RouteConfig = {
  label: "Reviews",
  icon: Star,
  rank: 10,
}

export default function ReviewsPage() {
  return <div>Reviews</div>
}
```

This creates a `/reviews` route and adds a "Reviews" item to the sidebar.

### The `config` export

The `config` export controls how the page appears in navigation:

| Property | Type | Description |
|----------|------|-------------|
| `label` | `string` | **Required.** Text shown in the sidebar menu |
| `icon` | `ComponentType` | Icon component (e.g. from `@medusajs/icons`) |
| `rank` | `number` | Sort order — lower numbers appear first |
| `nested` | `string` | Parent path for nested menu items |
| `translationNs` | `string` | i18n namespace for the label |
| `public` | `boolean` | If `true`, the route is accessible without authentication |

Pages without a `config` export are still routed but won't appear in the sidebar.

## Routing conventions

File paths map to URL routes automatically:

| File path | Route | Description |
|-----------|-------|-------------|
| `src/pages/page.tsx` | `/` | Root page |
| `src/pages/reviews/page.tsx` | `/reviews` | Static segment |
| `src/pages/reviews/[id]/page.tsx` | `/reviews/:id` | Dynamic segment |
| `src/pages/reviews/[[id]]/page.tsx` | `/reviews/:id?` | Optional dynamic segment |
| `src/pages/search/[*].tsx` | `/search/*` | Catch-all |
| `src/pages/(settings)/page.tsx` | Route grouping | Groups routes without adding a URL segment |
| `src/pages/dashboard/@sidebar/page.tsx` | Parallel route | Renders alongside parent |

## Navigation

Sidebar items are generated from pages that export a `config` with a `label`.

### Ordering

Use `rank` to control the order. Lower values appear higher:

```tsx
// src/pages/orders/page.tsx
export const config: RouteConfig = {
  label: "Orders",
  icon: ShoppingCart,
  rank: 1,
}

// src/pages/products/page.tsx
export const config: RouteConfig = {
  label: "Products",
  icon: Tag,
  rank: 2,
}
```

### Nested menus

Use `nested` to group pages under a parent:

```tsx src/pages/settings/shipping/page.tsx
export const config: RouteConfig = {
  label: "Shipping",
  nested: "/settings",
}
```

This places "Shipping" as a child item under `/settings` in the sidebar.

## Branding

Customize the panel's name and logo by passing options to `mercurDashboardPlugin` in `vite.config.ts`:

```typescript apps/vendor/vite.config.ts
mercurDashboardPlugin({
  medusaConfigPath: '../../packages/api/medusa-config.ts',
  name: 'My Marketplace',
  logo: 'https://example.com/logo.svg',
})
```

Both `name` and `logo` are optional. `logo` accepts a URL — the value is passed directly to an `<img src>` attribute without asset processing.

## Replacing components

You can replace specific layout components — the main sidebar, settings sidebar, or topbar actions — by providing component paths under the `components` option:

```typescript apps/vendor/vite.config.ts
mercurDashboardPlugin({
  medusaConfigPath: '../../packages/api/medusa-config.ts',
  components: {
    MainSidebar: 'components/custom-sidebar',
    SettingsSidebar: 'components/custom-settings-sidebar',
    TopbarActions: 'components/custom-topbar-actions',
  },
})
```

Each path is resolved relative to `src/` in the panel app. Each file must have a default export:

```tsx src/components/custom-topbar-actions.tsx
export default function CustomTopbarActions() {
  return <div>{/* your custom actions */}</div>
}
```

Available slots:

| Key | Description |
|-----|-------------|
| `MainSidebar` | Replaces the primary navigation sidebar |
| `SettingsSidebar` | Replaces the sidebar inside the settings area |
| `TopbarActions` | Replaces the action area in the top navigation bar |

## Internationalization

**1. Create translation resources**

```typescript src/i18n/index.ts
export default {
  en: {
    reviews: {
      title: "Reviews",
      description: "Manage product reviews",
    },
  },
  de: {
    reviews: {
      title: "Bewertungen",
      description: "Produktbewertungen verwalten",
    },
  },
}
```

**2. Reference the namespace in your page config**

```tsx src/pages/reviews/page.tsx
export const config: RouteConfig = {
  label: "reviews.title",
  icon: Star,
  translationNs: "reviews",
}
```

**3. Set the default language**

Pass `i18n.defaultLanguage` to `mercurDashboardPlugin` in `vite.config.ts`:

```typescript apps/vendor/vite.config.ts
mercurDashboardPlugin({
  medusaConfigPath: '../../packages/api/medusa-config.ts',
  i18n: { defaultLanguage: 'pl' },
})
```
